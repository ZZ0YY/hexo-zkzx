## 升学e网通刷课助手：让刷卷变得如此简单

作为一名升学e网通（ewt360.com）的用户，你是否曾经面对大量的独立试卷感到头疼？每一张都要手动填写、提交，耗费大量时间和精力。今天为大家介绍一款完全免费的开源神器——**升学e网通刷课助手**，它能在浏览器中一键扫描试卷、获取答案并自动提交，轻松实现满分完成。

![升学e网通刷课助手侧边栏界面](https://alist.20080601.xyz/d/cfbed/telegram/油猴脚本/1784587927154_屏幕截图_2026-07-21_065146.png)

### 什么是升学e网通刷课助手？

升学e网通刷课助手是一款运行在 Tampermonkey（油猴）上的用户脚本，专门针对升学e网通（ewt360.com）网站设计。安装后，它会自动在网页右下角添加一个粉紫色的悬浮按钮，点击即可展开一个萌系风格的侧边栏面板，所有操作都在这个面板中完成。

这款脚本完全免费开源，源代码托管在 GitHub 上，采用 MIT 许可证发布，你可以放心使用。项目使用 TypeScript 开发，如果你有编程基础，还可以自行修改构建。

### 核心功能一览

#### 1. 一键扫描独立试卷 📋

打开脚本面板后，点击「刷新列表」按钮，脚本会自动调用 EWT 官方 API，扫描你账号下的所有作业，并从中筛选出 contentTypeName 包含「试卷」的独立试卷。试卷会按照作业分组展示，每个作业下的试卷数量一目了然。

更智能的是，如果你当前正在作业详情页（URL 中包含 `homeworkId`），脚本会自动识别并只扫描当前作业的试卷，不会干扰其他作业。

#### 2. 自动获取答案并填写提交 🤖

这是脚本的核心功能。在试卷列表中，每张试卷右侧都有一个绿色的「刷」按钮，点击后脚本会：

1. **调后端获取标准答案** — 将你的试卷 ID、作业 ID 和登录凭证发送至后端服务器
2. **验证身份** — 后端使用你的 EWT Token 调用官方接口验证有效
3. **获取标准答案** — 后端返回完整的标准答案数据
4. **在浏览器中填卷** — 选择题直接填写正确的选项，非选择题利用系统自批机制实现满分
5. **提交并自批** — 自动提交试卷并完成批改

整个过程全自动完成，你只需要点击一个按钮，剩下的交给脚本处理。完成后会有显眼的「✅ 完成」提示，同时桌面上会弹出 Tampermonkey 通知。

#### 3. 试卷列表本地缓存 📦

刷新的试卷列表会自动缓存在浏览器本地（有效期5分钟），即使刷新页面或关闭面板，再次打开也不需要重新扫描，大大提升了使用体验。

#### 4. 每日限制与防滥用机制 ⚠️

为了防止接口被恶意滥用，后端设置了**每个 IP 每天最多刷 20 张试卷**的限制。这是后端硬性限制，无法通过修改脚本绕过。对于正常学生用户来说，这个额度完全足够日常使用。

### 技术架构与原理

对于感兴趣的用户，这里简单介绍一下脚本的技术实现：

```
油猴脚本（浏览器）                   后端（服务器）
     │                                   │
     ├─ 扫描试卷列表 ←── EWT API ────────┤
     │                                   │
     ├─ POST /api/get-answers ──────────→├─ 验证 EWT Token
     │  { paper_id, homework_id,         ├─ 后端拿答案
     │    ewt_token }                    ├─ 返回标准答案
     │  ←───────────────────────────────┤
     │                                   │
     ├─ 前端填卷 → 交卷 → 自批           │
     │  (EWT API，使用用户自己的 token)    │
```

项目使用 **TypeScript** 编写，通过 **Vite** 构建，源代码结构清晰：

- `src/main.ts` — 脚本入口，负责页面检测、事件绑定、流程编排
- `src/utils/token.ts` — 从 Cookie 中提取 EWT 登录凭证
- `src/utils/ewtApi.ts` — 对 `GM_xmlhttpRequest` 的封装，用于跨域请求
- `src/services/paperScanner.ts` — 调用 EWT 官方接口，扫描作业和试卷
- `src/services/paperFiller.ts` — 获取答案后在浏览器中自动填卷、交卷、自批
- `src/components/SidePanel.ts` — 完整的萌系 UI 侧边栏组件

如果你想自行修改并构建：

```bash
git clone https://github.com/ZZ0YY/EWT-TOOL.git
cd EWT-TOOL
npm install
npm run build
```

构建产物在 `dist/ewt-paper-helper.user.js`。

### 常见问题 FAQ

#### Q：脚本安全吗？会不会被盗号？

脚本仅通过 EWT 官方 API 进行操作，你的登录凭证（Token）会传给后端用于验证身份。后端仅用 Token 调用 EWT 官方接口验证有效性，**不会存储或用于其他用途**。所有刷卷操作都在你自己的浏览器中完成，代码完全开源，可自行审查。

#### Q：为什么点击刷按钮后没有反应？

请检查：
- 是否已经登录 ewt360.com？
- Tampermonkey 是否已启用本脚本？
- 是否在脚本弹出跨域请求权限时选择了「允许」？
- 如果使用了广告拦截器，请尝试关闭后重试

#### Q：提示「获取答案失败」是什么原因？

可能原因：
- 当前试卷后端暂无答案数据
- IP 已达每日 20 张的限制
- 后端服务临时故障（可稍后再试）

#### Q：每日 20 张不够用怎么办？

20 张/天的限制是后端为了防止恶意滥用设置的保护机制。对于正常使用来说，一天完成 20 张独立试卷已经相当可观。如果确实有大量需求，建议合理规划，分多天完成。

#### Q：脚本支持移动端吗？

脚本主要针对桌面端浏览器设计。移动端浏览器大多不支持 Tampermonkey 扩展，建议在电脑上使用 Chrome 或 Edge 浏览器安装使用。

#### Q：如何更新脚本？

- 从 Greasy Fork 安装的脚本会自动检查更新
- 也可以手动在 Tampermonkey 管理面板中点击「检查更新」
